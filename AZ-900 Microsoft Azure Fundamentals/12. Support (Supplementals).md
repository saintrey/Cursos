# 12. Support (Supplementals) - Soporte y Ayuda

## Resumen del Capítulo
Azure proporciona múltiples opciones de soporte para ayudarte. Este capítulo cubre planes de soporte Azure, canales de ayuda, Microsoft Learn, y recursos comunitarios. **Nota**: Este capítulo es SUPPLEMENTAL, no es parte del examen AZ-900 actual (refreshed May 2022).

## Temas Principales

### 1. Azure Support Plans (Planes de Soporte)

#### a) Free Support (Gratuito)
- **Incluido** con todas las suscripciones Azure
- Acceso a documentación oficial
- Community support
- Service health monitoring
- **Limitación**: No hay soporte ticket directo

#### b) Developer Support (~$25/mes)
- Para: No-production environments
- Response time: < 24 horas (durante horas trabajo)
- Email-based support
- **Casos válidos**: 
  - Technical guidance
  - Breaking changes

#### c) Standard Support (~$100/mes)
- Para: Production workloads
- Response times:
  - Non-critical: < 24 horas
  - System impaired: < 4 horas
  - Production impaired: < 1 hora
- Phone y email support

#### d) Professional Direct Support (~$1000/mes)
- Para: Business-critical applications
- Response times:
  - Non-critical: < 15 minutos
  - System impaired: < 15 minutos
  - Production down: < 15 minutos
- Phone support 24/7
- Assigned Technical Account Manager (TAM)
- Proactive guidance

### 4. Status Page y Health Dashboards

#### a) Azure Status Page
- Real-time info sobre servicios
- Shows: Operational, Degraded, Down
- Geographic information
- Historical data

#### b) Service Health Portal
- In Azure portal
- Personalized health view
- Advisories específicos a tu resources
- Planned maintenance schedule

#### c) Planned Maintenance
- Azure publica schedule adelantado
- Notificaciones por email
- Recommended: Schedule work around it

### 5. Advisory Alerts

#### a) Service Health Alerts
- Problemas detectados con tus resources
- Email notifications
- Action items recomendados
- Severity levels

#### b) Azure Advisor
- **Recommendations automáticas**:
  - Cost optimization
  - Security improvements
  - Performance optimization
  - Reliability
- Free para todos los suscriptores

### 6. Support Case Management

#### a) Crear Support Ticket
```
Azure Portal:
  → Help + Support
  → New Support Request
  
Enter:
  - Issue type
  - Severity
  - Subscription
  - Description
  - Contact info
```

#### b) Severity Levels
```
Severity 1 (Critical):
  - Production down
  - No workaround
  - Response: Immediate

Severity 2 (High):
  - Moderate impact
  - Workaround available
  - Response: 1 hour

Severity 3 (Medium):
  - Low impact
  - Minimal workaround
  - Response: 4 hours

Severity 4 (Low):
  - Information request
  - No impact
  - Response: 24 hours
```

### 9. Contactarme Scenarios

#### a) Tengo Pregunta Técnica
```
1era intentar: Microsoft Learn, Docs
2da intentar: Stack Overflow, Community Forum
3era intentar: Support ticket (if paid plan)
```

#### b) Tengo Bug
```
1. Reproduce consistentemente
2. Gather diagnostics
3. Submit support ticket with:
   - Details del bug
   - Repro steps
   - Error logs
```

#### c) Quiero Aprender
```
1. Microsoft Learn (free)
2. Azure Documentation
3. Hands-on labs
4. Sandboxed environment
```

#### d) Tengo Production Issue
```
1. Check Azure Status page
2. Escalada si needed
3. Call support hotline si Premium Direct plan
4. Provide reproduction
```

### 10. Diferencias en Support Plans

| Aspecto | Free | Developer | Standard | Professional |
|--------|------|-----------|----------|-------------|
| **Costo** | $0 | $25 | $100 | $1000 |
| **Tickets** | Comunidad | Email | Email+Phone | 24/7 Phone |
| **Response** | N/A | 24h | 1-4h | 15min |
| **TAM** | No | No | No | Sí |
| **Para** | Learn | Dev | Production | Mission-critical |

## Flujo: Buscar Ayuda

```
¿Pregunta básica?
  ↓ YES → Microsoft Docs / Learn
  ↓ NO
¿Pregunta técnica específica?
  ↓ YES → Stack Overflow / Community
  ↓ NO
¿Problema con servicio Azure?
  ↓ YES → Check Status page
  ↓ NO
¿Tienes plan de soporte pagado?
  ↓ YES → Create support ticket
  ↓ NO  → Escalada si needed
```
