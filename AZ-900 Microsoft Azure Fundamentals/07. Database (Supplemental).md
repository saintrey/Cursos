# 07. Database (Supplemental) - Base de Datos

## Resumen del Capítulo
Este capítulo cubre los servicios de base de datos disponibles en Azure. Incluye Cosmos DB (NoSQL global), Azure SQL (relacional managed), y alternativas open-source como MySQL y PostgreSQL. **Nota**: Este capítulo es SUPPLEMENTAL, no es parte del examen AZ-900 actual (refreshed May 2022).

## Temas Principales

### 1. Introducción: Qué es una Database

#### a) Problemas del Almacenamiento Simple
```
Sin Database:
- Datos en archivos sueltos
- Documentos en montón
- Difícil encontrar algo específico
- Necesita búsqueda manual

Con Database:
- Automáticamente organizado
- Indexado y ordenado
- Búsquedas rápidas y complejas
- Obtén datos en formato exacto que quieres
```

#### b) Poder de Databases
- Almacenamiento eficiente
- Recuperación rápida
- Búsquedas complejas
- Relaciones entre datos
- Queries potentes

### 2. Cosmos DB

#### a) Definición
- **Globally distributed** managed database
- Super rápido, super fácil de scaler
- NoSQL (no relacional)
- Desarrollado específicamente para cloud

#### b) Superpower: Global Distribution

**El Problema**:
```
Database en Australia
Usuarios en Paris, USA, Africa
Todos necesitan acceso rápido
Latencia alta para usuarios lejanos
```

**Solución: Cosmos DB**
```
Data en múltiples regiones automáticamente
Usuarios acceden data cercana
Cambios se replican globalmente
Todo sincronizado por Azure
```

#### c) Características Principales

**1. Global Scale - Con Un Click**
- Add región → Click botón en portal
- Azure maneja replicación automáticamente
- Expande a Australia, Japan, Europa, etc.

**2. Single-Digit Millisecond Latency**
- <10ms latencias (más o menos que un parpadeo)
- Anywhere en el mundo
- Usuarios muy contentos

**3. Auto-Scaling**
- Demand sube → Cosmos escala automáticamente
- Demand baja → Escala down
- Sin límite conocido de usuarios
- Pagas solo por lo usado

**4. Developer-Friendly**
- Múltiples SDKs: .NET, Java, Node.js, Python
- APIs nativas para diferentes platforms
- SQL, MongoDB, Cassandra soportados
- Fácil integración

#### d) Pitfall: Costos
- Puede ser caro conforme escala
- Monitorea costos cuidadosamente
- Azure tiene herramientas de cost management

### 3. Azure SQL

#### a) Definición
- **Managed database service** basado en Microsoft SQL
- SQL es producto Microsoft desde 1989 (muy stable)
- PaaS: Azure administra hardware/infraestructura

#### b) Ventajas
- **On-Premises Migration**: Move SQL Server → AZURE sin cambios
- **Machine Learning Insights**: Recomendaciones automáticas para performance
- **Security**: Aprovecha equipo Azure de seguridad
- **Scalability**: Up to 100TB
- **HA**: Restore huge databases en minutos
- **Total Cost of Ownership**: Reduce dramáticamente vs on-premises

#### c) Dos Variantes

**1. SQL Database**
- Más features tradicionales SQL
- Para aplicaciones modernas

**2. SQL Managed Instance**
- Brinda gap entre on-premises SQL Server y Azure SQL Database
- Para legacy features support
- Más familiar para administradores SQL on-premises

#### d) Ambos Type Benefician De
- Recomendaciones automáticas
- Monitoreo de health
- Escalabilidad cloud
- Seguridad integrada

### 4. Azure Database for MySQL

#### a) Definición
- **Managed service** para MySQL (community-based)
- Open-source, relacional
- MySQL #1 en internet (millones de databases)

#### b) Ventajas
- **No Infrastructure**: Azure maneja servers
- **Multi-language**: Cualquier lenguaje
- **WordPress Support**: WordPress + MySQL en Azure
- **High Availability**: Scale rápidamente
- **Azure Security Features**: Amenazas avanzadas, monitoreo, identidad
- **Fully Managed**: Patching automático, backups, monitoreo

#### c) Casos de Uso
- Web applications con Machine Learning
- E-commerce
- Mobile applications (scale a millones)
- Digital marketing (traffic spikes)
- Finance (datos sensibles)
- Gaming (latency bajo)

### 5. Azure Database for PostgreSQL

#### a) Definición
- **Managed service** para PostgreSQL
- "Post-Ingres" → Nombre histórico
- Desde 1996, open-source
- macOS default database
- Huge user base

#### b) Características Únicas
- **Extensiones**: JSON, spatial, funcionalidades ricas
- **JSONB Support**: Binary JSON format
- **Visual Studio Code Integration**: Desarrollo amigable
- **Horizontal Scaling**: Hyperscale → Cientos de nodes

#### c) Ventajas Azure PostgreSQL
- **No Rewrites**: Scale sin cambios de código
- **Performance Recommendations**: Automáticas
- **Monitoring**: Detecta eventos disruptivos
- **Fully Managed**: Patcheo, backups, monitoreo gratis

#### d) Industrias que Usan PostgreSQL
- **Finance**: Online transaction processing
- **Government**: GIS data (geometric)
- **Manufacturing**: Reliability crítica, failover automático, zero-downtime upgrades
- **Research**: Mathematical software (MATLAB, R)

### 6. Database Migration Services

#### a) Problema
```
Tengo database on-premises
Quiero mover a Azure
¿Cómo hago sin downtime y errores?
→ Azure Database Migration Services
```

#### b) Solución
- **Herramienta dedicada** para migraciones
- Move SQL Server on-premises → Azure SQL Managed Instance
- Move MySQL, PostgreSQL, non-Microsoft → Azure
- Comprehensive documentation
- Step-by-step guidance

#### c) Capacidades
- Migra SQL Server on-premises → Azure sin cambios
- Migra MySQL on-premises → Azure MySQL
- Migra PostgreSQL on-premises → Azure PostgreSQL
- Detalladas guides para migraciones non-Microsoft
- Integración con Azure Migrate para full datacenter migration

#### d) Beneficio: Simplify
```
Before DMS:
- Manual migration
- Risk de downtime
- Errores posibles
- No documentation

Con DMS:
- Automated
- Minimal downtime
- Guías paso-step
- Supported tools
```

### 8. Conceptos Database Importantes

#### a) Relational vs NoSQL

**Relational (SQL)**:
- Datos en tablas
- Relaciones entre tablas
- Consultas complejas
- Transactions
- Ejemplo: Clientes ↔ Órdenes

**NoSQL (Cosmos)**:
- Datos en documentos
- Flexible schema
- Escalable horizontalmente
- Ejemplo: Documento JSON con todo adentro

#### b) Escalabilidad

**Vertical** (Scale Up):
- Más poder a máquina existente
- Limitado (máximo de poder)

**Horizontal** (Scale Out):
- Múltiples máquinas
- Unlimited (agregas más máquinas)
- PostgreSQL con Hyperscale = Cientos de nodes
