# 01. Introducción

## Resumen del Capítulo
La introducción al curso AZ-900 cubre la estructura general del examen, las herramientas fundamentales para interactuar con Azure y cómo prepararse para la certificación Microsoft Azure Fundamentals.

## Temas Principales

### 1. Visión General del Curso
- **Objetivo**: Proporcionar los conocimientos necesarios para aprobar el examen AZ-900
- **Certificación**: Primer paso en la carrera profesional con Azure
- **Beneficios de la certificación**:
  - Credibilidad profesional en el mercado
  - Acceso a red de profesionales certificados
  - Posibilidad de aumentar salario
  - Mayor confianza en aplicación de conocimientos

### 2. Estructura del Curso
Los temas principales cubiertos son:
- Cloud Concepts (Conceptos de Nube)
- Azure Architecture (Arquitectura de Azure)
- Compute (Computación)
- Networking (Redes)
- Storage (Almacenamiento)
- Database (Base de Datos)
- Authentication and Authorization (Autenticación y Autorización)
- Solutions (Soluciones)
- Security (Seguridad)
- Pricing (Precios)
- Support (Soporte)

### 3. Herramientas de Azure

#### a) Azure Portal
- Forma más común de interactuar con Azure
- Interfaz web intuitiva
- Acceso a todos los recursos
- Personalización con dashboards
- Control de acceso granular
- Gestión de costos integrada

#### b) Azure CLI
- Interfaz de línea de comandos
- Comandos de texto estructura lógica
- Multiplataforma (Windows, Mac, Linux)
- Ideal para automatización
- Confiable para scripting

#### c) PowerShell
- Herramienta para scripts avanzados
- Usa cmdlets (pequeños programas)
- Integración con Azure Resource Manager
- Preferido por profesionales IT
- Lenguaje propio de programación

#### d) Azure Cloud Shell
- Shell interactivo accesible desde navegador
- Opción Bash (CLI) o PowerShell
- Accesible desde web, móvil o cualquier dispositivo
- Autenticado y 100% seguro
- Herramientas comunes incluidas
- Almacenamiento propio para persistencia

#### e) Azure Mobile Apps
- Disponible para iOS y Android
- Información rápida en movilidad
- Alertas y estado del sistema
- Acceso a Cloud Shell desde el teléfono
- Sincronizado con Azure Resource Manager

### 4. ARM Templates (Plantillas de Azure Resource Manager)
- **Definición**: Forma de describir qué recursos crear/gestionar en Azure
- **Formato**: JSON
- **Ventajas**:
  - Idempotentes (mismo resultado al aplicar múltiples veces)
  - Automatización y evita errores humanos
  - Control de versiones
  - Reutilizable
  - Declarativo (especificas qué quieres, no cómo)
  - Replicación y consistencia

### 5. Azure Advisor
- Servicio gratuito que viene con Azure
- Proporciona recomendaciones para:
  - Mejorar disponibilidad de recursos
  - Reducir costos
  - Aumentar confiabilidad
  - Mejorar seguridad
  - Excelencia operativa
- Analiza infraestructura automáticamente

### 6. Creación de Cuenta Azure Gratuita
- Require email y verificación de número de teléfono
- Necesita tarjeta de crédito (para verificación, NO cobra sin autorización)
- Incluye crédito inicial gratuito
- Acceso a miles de imágenes y configuraciones

## 📚 Conclusión y Síntesis Profunda - Capítulo 01: Introducción

### ¿Qué Aprendiste Realmente?

Este capítulo introductorio te presentó los **pilares fundamentales** sobre los cuales se construye toda tu experiencia con Azure. Al completar este capítulo, deberías entender que Azure no es solo un producto de software, sino un **ecosistema completo** de herramientas, servicios y convenciones diseñado para modernizar cómo construimos, desplegamos y gestionamos infraestructura en la nube.

### Por Qué Es Importante

Muchos principiantes cometen el error de intentar "saltarse" la introducción. **No hagas eso.** Esta sección es crítica porque:

1. **Establece el vocabulario**: Conocer las herramientas (Portal, CLI, PowerShell, Cloud Shell) es tu primer paso. Sin estas herramientas, no puedes interactuar con Azure. Cada una tiene un propósito específico:
   - **Portal**: Para explorar y entender visualmente (perfecto para aprender)
   - **CLI/PowerShell**: Para automatización y scripting (necesario para producción)
   - **Cloud Shell**: Para acceso rápido sin instalación (ideal para empezar)

2. **ARM Templates son la base de la automatización**: Cuando comprendas que ARM Templates es simplemente JSON que describe "qué quieres," empezarás a ver el poder de Infrastructure as Code. Esto significa que puedes versionar tu infraestructura como si fuera código, reproducirla perfectamente cada vez, y colaborar con otros desarrolladores.

3. **Azure Advisor es tu consultor gratuito**: Este servicio es el que muchas empresas pagan consultores externos para obtener. Azure Advisor automáticamente revisa tu infraestructura y dice "oye, aquí puedes ahorrar dinero" o "esta VM podría tener mejor disponibilidad si hace esto." Es como tener un experto mirando tu setup 24/7.

### Cómo Se Conecta Con Todo Lo Demás

Imagina que Azure es una casa grande:

```
INTRO (Cimientos y herramientas)
    ↓
CLOUD CONCEPTS (Los principios)
    ↓
ARCHITECTURE (El plano de la casa)
    ↓
COMPUTE/NETWORKING/STORAGE (Las habitaciones)
    ↓
SECURITY (Las cerraduras y alarmas)
    ↓
MANAGEMENT (Cómo mantener todo funcionando)
```

Sin entender los cimientos (esta introducción), todo lo demás se construye sobre arena. Cuando aprendas sobre VMs (Compute), necesitarás recordar que puedes crear ARM Templates para ellas. Cuando aprendas sobre Seguridad, usarás el Portal para implementarla. Cuando uses Azure Advisor, te dirá cómo optimizar esas VMs.

### Caso de Uso Práctico

Imagina que eres una startup newtech:

1. **Día 1**: Creas una cuenta Azure gratuita (aprendiste cómo hacerlo)
2. **Día 2**: Exploras el Azure Portal (entiendes la interfaz)
3. **Día 3**: Tu CTO decide "necesitamos replicar esto en otra región para disaster recovery"
4. **Día 4**: En lugar de hacer click 500 veces, usas ARM Template (que copiaste del primer deployment)
5. **Día 5**: Azure Advisor dice "oye, esas VMs pueden usar reserved instances y ahorran 40%"
6. **Día 6**: Haces el cambio mediante PowerShell (mientras te tomas un café)

Todo eso que acabas de leer depende de entender esta introducción.

### Lo Que Deberías Hacer Ahora

No solo memorices qué es Portal, CLI y PowerShell. **Prueba cada uno**:

- Abre Azure Portal y haz click alrededor (sin miedo, es free trial)
- Abre Cloud Shell dentro del Portal e intenta un comando: `az vm list`
- Lee una ARM Template simple (busca en GitHub) y trata de entender la estructura JSON

Este "learning by doing" es 1000% mejor que solo leer.

### Conexión con el Examen AZ-900

El examen AZ-900 probablemente NO te preguntará "¿Cuál es el comando exacto de CLI?" Pero SÍ te preguntará cosas como:
- "¿Cuál es la mejor herramienta para automatizar deployment?" (ARM Templates)
- "¿Dónde ves recomendaciones de costo automáticamente?" (Azure Advisor)
- "¿Cuándo usarías PowerShell vs Portal?" (Automatización vs Exploración)

Estos conceptos son **sutil pero crítico** para aprobar el examen y trabajar en Azure profesionalmente.
