# 10. Security (Seguridad)

## Resumen del Capítulo
Security es crítica en Azure. Este capítulo cubre defensa en profundidad (Defense in Depth), Azure Security Center, protección de red, encriptación, y mejores prácticas de seguridad. Este capítulo **SÍ es parte** del examen AZ-900.

## Temas Principales

### 1. Defense in Depth (Defensa en Profundidad)

#### a) Concepto
- **Múltiples capas de seguridad**
- No confiar en una sola línea de defensa
- Si attacker bypasses 1 capa, quedan otras

#### b) Capas de Seguridad (De fuera hacia dentro)

```
EXTERIOR
    ↓
1. PHYSICAL SECURITY (Datacenters)
   - Físicamente protegido
   - Acceso limitado
   
2. PERIMETER SECURITY (DDoS protection)
   - Azure DDoS Protection
   - Bloquea ataques masivos
   
3. NETWORK SECURITY (Firewalls, NSGs)
   - Network Security Groups
   - Azure Firewall
   - VPN/ExpressRoute
   
4. APPLICATION SECURITY
   - Input validation
   - Web Application Firewall
   - Secure coding
   
5. DATA SECURITY (Encryption)
   - Encryption at rest
   - Encryption in transit
   - Key Management (Key Vault)
   
6. IDENTITY & ACCESS
   - Azure AD
   - MFA
   - RBAC
   
7. LOGGING & MONITORING
   - Azure Monitor
   - Security Center
   - Audit logs
   
INTERIOR (Tu responsabilidad)
```

#### c) Beneficio
- Attacker necesita romper MÚLTIPLES capas
- Complexity = Deterrent

### 2. Network Security Groups (NSGs)

#### a) Definición
- **Firewall virtual** para subnets y NICs
- Controla incoming/outgoing traffic
- Basado en reglas

#### b) Reglas NSG
```
Inbound Rules:
  - Quién puede ENTRAR a la subnet/VM
  - Basado en: Source IP, port, protocol
  
Outbound Rules:
  - Quién puede SALIR de la subnet/VM
  - Similar a inbound
```

#### c) Ejemplos de Reglas
```
Allow HTTP incoming on port 80
  → Usuarios pueden acceder website

Allow SSH from 10.0.0.0/24 only
  → Solo computadoras en subnet específica

Deny ALL outbound except DNS
  → VM solo puede hablar a DNS servers
```

### 3. Azure Firewall

#### a) Definición
- **Managed firewall service**
- Completa protección de red
- Basada en Cloud

#### b) Características
- Stateful firewall (mantiene estado)
- High availability integrada
- Logging y monitoring
- Protección DDoS básica

#### c) Diferencia: NSG vs Firewall
```
NSG:
  - Layer 4 (Transport)
  - Subnet/NIC level
  - Más granular

Azure Firewall:
  - Layer 7 (Application)
  - Network level
  - Inspección profunda
```

### 4. DDoS Protection

#### a) Qué es DDoS
- **Distributed Denial of Service**
- Ataques intentan sobrecargar servicio
- Millones de requests simultáneamente
- Servicio cae

#### b) Azure DDoS Protection
- **Standard**: Básico, siempre activo
- **Premium**: Avanzado, detection + mitigation
- Protege aplicaciones automáticamente

### 5. Azure Security Center

#### a) Definición
- **Central security dashboard** para Azure
- Monitorear seguridad de todos tus resources
- Recomendaciones automáticas
- Compliance tracking

#### b) Funcionalidades
- **Threat Detection**: Detecta ataques potenciales
- **Vulnerability Assessment**: Identifica debilidades
- **Recommendations**: Acciones para mejorar
- **Incidents**: Alertas en tiempo real

#### c) Scoring de Seguridad
- Calificación 0-100
- Basado en implementación de mejores prácticas
- Tareas específicas para mejorar score
- Benchmarking contra industry standards

### 6. Encryption (Encriptación)

#### a) Tipos de Encriptación

**1. Encryption at Rest**
- Data almacenado está encriptada
- En disco, en database
- Clave necesaria para leer
- Default en muchos servicios Azure

**2. Encryption in Transit**
- Data en movimiento está encriptada
- Entre client y server
- SSL/TLS típicamente
- HTTPS vs HTTP
- Protege contra interception

**3. End-to-End Encryption**
- Encriptada desde origen hasta destino
- Incluso Azure no puede leer
- Usuario controla claves

#### b) Azure Key Vault
- **Manages encryption keys**
- Almacena secrets, passwords, certificates
- Access control integrado
- Auditing completo
- Integración con otros servicios

### 7. Azure Information Protection

#### a) Definición
- **Data classification + protection**
- Categoriza qué tan sensible es data
- Aplica protección automáticamente

#### b) Etiquetas de Clasificación
```
Public:   Información pública, sin restricción
Internal: Solo para empleados
Confidential: Sensible, acceso restringido
Secret:   Highest sensitivity, muy pocas personas
```

#### c) Protecciones
- Encriptación automática
- Watermarks visuales
- Restricciones de acceso
- Expiration automática

### 8. Compliance y Governance

#### a) Azure Policy
- **Enforce compliance** en todo Azure
- Define reglas y estándares
- Automáticamente aplica
- Previene configuraciones inseguras

#### b) Ejemplos de Policies
```
"VMs deben tener MFA"
"Data debe estar encrypted"
"Solo ciertos regions permitidos"
"Tags requeridos en todos resources"
```

#### c) Azure Blueprints
- **Pre-defined environments** completos
- Implementan compliance automáticamente
- Reutilizable
- Versionado

### 9. Advanced Threat Protection

#### a) Azure Advanced Threat Protection (ATP)
- Detecta ataques sofisticados
- Basado en machine learning
- Identifica patrones anormales
- Alertas in real-time

#### b) Application Security (ASC)
- Application Gateway Web Application Firewall
- Protección contra: XSS, SQL injection, etc
- Layer 7 protection

### 10. Auditing y Compliance

#### a) Azure Audit Logs
- Registro de ACCIÓN en Azure
- "Quién hizo QUÉ cuándo"
- Compliance requirement
- Retenido por 90 días (extendible)

#### b) Azure Monitor
- Monitoring y alerting completo
- Logs de aplicaciones
- Performance metrics
- Behavior analysis

### 12. Best Practices Security

✅ **SIEMPRE**:
1. Enable MFA (Multi-Factor Authentication)
2. Use strong passwords o passwordless
3. Encrypt data (at rest + in transit)
4. Monitor y audit everything
5. Principle of Least Privilege
6. Keep software updated
7. Regular backups
8. Defense in Depth
9. Security awareness training
10. Regular security reviews

❌ **NUNCA**:
1. Share credentials
2. Store passwords en plain text
3. Disable security features
4. Use default passwords
5. Trust only perimeter
6. Ignore security alerts
7. Store secrets en code
