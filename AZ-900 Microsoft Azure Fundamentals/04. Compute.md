# 04. Compute (Computación)

## Resumen del Capítulo
Compute es el corazón de Azure. Cubre todas las formas de ejecutar aplicaciones en la nube: desde máquinas virtuales tradicionales hasta funciones serverless. Incluye IaaS (VMs), PaaS (App Services) y Serverless (Functions).

## Temas Principales

### 1. Virtual Machines (VMs - Máquinas Virtuales)

#### a) Definición
- **Computadora virtual** que tienes acceso EXCLUSIVO
- No posees/administras hardware físico (Azure sí)
- **IaaS**: Tú administras TODO excepto el hardware físico

#### b) Ventajas
- Acceso exclusivo a la máquina
- Escalabilidad: Crea/elimina rápidamente
- Herramientas Azure para administra miles de VMs
- Soporte multi-plataforma: Windows o Linux
- Recomendaciones automáticas de Azure

#### c) Casos de Uso
- Necesitas control total de la máquina
- Instalar aplicación específica
- Migrar infraestructura on-premises
- Cuando otros servicios (App Services) no encajan

#### d) Cuándo NO usar VMs
- Si App Services/PaaS cubre tu caso (menos mantenimiento)
- El mantenimiento es responsabilidad tuya (pain point)

#### e) Precios
- **Calculado por hora**
- Depende de CPU y RAM (a más, más caro)
- **Nuevas opciones**: GPUs especializadas
- GPU para: AI, Deep Learning

#### f) Características Azure para VMs
- **Availability Sets**: Grupo de VMs para alta disponibilidad
- **Blueprints**: Templates de cómo crear VMs (consistency)
- **Azure Advisor**: Recomendaciones automáticas para HA, seguridad, performance, costos

### 2. Scale Sets (Conjuntos de Escalado)

#### a) Definición
- **Grupo de VMs idénticas**, load-balanced
- Crea clones a partir de una VM base
- Auto-scaling: Agrega/elimina automáticamente

#### b) Analogía del Vaso de Agua
```
1 VM = 1 vaso a 50% capacidad (CPU 50%)
Si sube a 90%:
  ↓
Agrega otro VM automáticamente = 2 vasos a ~45% cada uno
Si baja la demanda:
  ↓
Elimina vasos automáticamente (no pagas por recursos no usados)
```

#### c) Características
- **Idénticas**: Todas las VMs en scale set son iguales
- **Load Balanced**: Distribuye tráfico automáticamente
- **Auto-Scaling**: Basado en reglas o cpu/memoria
- **Scheduled Scaling**: Escala a horas específicas

#### d) Beneficios
- Fácil gestión de múltiples VMs
- Consistency (todas iguales)
- Load balancing integrado
- **Alta Disponibilidad**: Si 1 VM falla, otras sirven
- Auto-scaling reduce costos
- Escalable hasta 1000 VMs en un scale set
- **Sin costo extra**: Pagas solo por VMs + storage + load balancers

#### e) Caso de Uso Real
- E-commerce: Tráfico sube noches/weekends
- Scale Sets automáticamente agrega VMs
- Cuando tráfico baja, elimina VMs
- Result: Mejor performance + costos optimizados

### 3. App Services

#### a) Definición
- **Platform as a Service (PaaS)** completamente managed
- Azure administra: servidores, red, storage, infraestructura
- Tú administras: tu lógica de negocio

#### b) Tres Tipos de App Services

**1. Web Apps**
- Websites y aplicaciones online típicas
- Soporta: .NET, Java, Node.js, PHP, Python (Windows) o .NET Core, Node.js, PHP, Ruby (Linux)
- Integración con herramientas Azure/Microsoft
- Auto-scaling y load balancing
- Fácil deployment

**2. Web Apps for Containers**
- Despliega aplicaciones containerizadas
- Containers = paquete de app + todas sus dependencias
- Portable: Funciona en cualquier lugar con container runtime
- Analogía: Nintendo Cartridge (todo lo que necesitas adentro)

**3. API Apps**
- Backend data services sin frontend gráfico
- Application Programming Interface
- Conecta otras apps programáticamente
- Lenguajes: C#, Python, PHP, Java
- Acceso seguro integrado

#### c) Ventajas App Services
- Fully managed: No worry about hardware
- Multi-lenguaje support
- Built-in auto-scaling + load balancing
- Fácil de usar

### 4. Azure Container Instances (ACI)

#### a) Qué es un Container
- **Self-contained unit** de software
- Incluye: código, programas, todas las dependencias
- **Portable**: Funciona en cualquier máquina con container runtime
- Analogía: Envío de paquetes en containers estándar vs. custom packaging
- Resultado: Consistencia entre ambientes

#### b) Analogía Detallada del Envío
```
Sin Containers (Tradicional):
  - Cada item necesita custom packaging
  - Lento, error-prone
  (Como los desarrolladores diciendo "funciona en mi máquina")

Con Containers:
  - Todo en container estándar
  - Funciona en cualquier lugar
  - Consistencia garantizada
```

#### c) Beneficios Containers
- **Manejabilidad de Dependencias**: Todo adentro del container
- **Menos Overhead**: Comparado con VMs
- **Portabilidad**: Deploya en cualquier lugar
- **Efficiency**: Ciclo desarrollo/deployment más rápido
- **Scaling**: Más fácil escalar containers que VMs
- **Consistency**: Mismo comportamiento everywhere

#### d) Azure Container Instances (ACI)
- Servicio Azure para hostear/correr containers
- Sin administrar VMs
- On-demand: Crea container solo cuando lo necesites, después apaga
- Perfecto para: Procesamiento data bajo demanda
- Acceso: Portal, CLI, PowerShell

### 5. Azure Kubernetes Service (AKS)

#### a) Qué es Kubernetes
- **Orchestration system** open-source por Google (nombre griego: "govern")
- Automatiza: deployment, scaling, management de containers
- **No es Azure-specific**: Kubernetes es universal

#### b) Funcionalidades Kubernetes
- **Automatic Deployment**: Despliega instancias automáticamente
- **Automatic Scaling**: Monitorea carga, escala según sea necesario
- **Replicable Architecture**: Reutiliza setups probados

#### c) Azure Kubernetes Service (AKS)
- **Managed** Kubernetes por Azure
- Includes: All Kubernetes features + Azure services
- **Cluster**: Set de máquinas (nodes) que corren containers
- **Pod**: Grupo de 1+ containers con storage/network compartidos
- Integración: Identity Management, elastic provisioning, VS Code, global reach

#### d) Azure Container Registry (ACR)
- Almacena container images (como un registro)
- Cuando AKS/ACI necesita imagen, viene de ACR
- Seguridad Azure integrada
- Es como "app store" de containers

#### e) Caso de Uso Kubernetes
```
App procesa órdenes de clientes → Container
Más órdenes → Kubernetes automáticamente crea more Pods
Fewer órdenes → Kubernetes elimina Pods innecesarios
```

### 6. Azure Virtual Desktop (AVD)

#### a) Definición
- Windows 10 completamente virtualizado 100% en Azure
- Acceso desde: Windows, Mac, iOS, Linux, smartphones
- No corre en tu máquina, corre en Azure

#### b) Casos de Uso
- Empleados remotos: Rápido acceso a Windows en cloud
- Usuarios on-premise o on-premises+cloud
- Multi-user: Múltiples usuarios en misma instancia

#### c) Ventajas
- **Licensing**: Reutiliza Windows 10 existentes (reduce costos)
- **Multi-Session**: Múltiples usuarios = instancia
- **Anywhere Access**: De qualquier dispositivo, ubicación
- **Data Security**: Data en Azure, no en dispositivo
- **Integration**: Azure security features (MFA, RBAC, Policies)

### 7. Azure Functions (Serverless)

#### a) Definición
- **Smallest + largest** compute service Azure
- **Función única** de computación
- Se ejecuta cuando su URL es llamada (invoked)
- Después de completarse: Se muere (sin persistencia)
- **TRUE Serverless**: No administras servers en absoluto

#### b) Arquitectura
```
User hits function URL
    ↓
Function triggered
    ↓
Executes once
    ↓
Talks to other services / returns response
    ↓
Function ends (dies)
```

#### c) Caso de Uso: FunnyFerret Business
```
Usuario sube imagen de ferret graciosa
    ↓
Azure Function:
  - Comprime imagen
  - Convierte a formato web
  - Almacena en base de datos
    ↓
Function completa y muere
```

#### d) Ventajas Functions
- **Costo**: Pagas solo cuando se ejecuta (no running 24/7)
- **Resilience**: Si 1 function falla, otros no se ven afectados
- **Isolation**: Cada función aislada (risk spread)

#### e) ACG Infraestructura
- **Todo serverless**: Cero VMs en ACG
- Permite crecimiento flexible
- Sirve cursos a audiencia masiva
- Trend continuo de serverless

## Comparación de Compute Services

| Servicio | Tipo | Control | Mantenimiento | Scaling | Costo |
|----------|------|---------|-------------|---------|-------|
| VM | IaaS | Alto | Alto | Manual | Por hora |
| Scale Sets | IaaS | Alto | Alto | Auto | Por hora |
| App Services | PaaS | Medio | Bajo | Auto | Según plan |
| Containers | Micro | Medio | Medio | Auto | Según uso |
| Kubernetes | Orquestr | Alto | Medio | Auto | Según recursos |
| Functions | Serverless | Bajo | Ninguno | Auto | Por ejecución |

## Puntos Clave para el Examen

✅ **Recuerda**:
- **VMs**: Control total, mantenimiento total
- **Scale Sets**: Múltiples VMs idénticas, auto-scaling
- **App Services**: PaaS, managed, fácil
- **Containers**: Self-contained, portable
- **AKS**: Orquestar many containers
- **Functions**: True serverless, paga por uso
- **Serverless ≠ Sin servidores** (solo no los administras)
- Escalabilidad = Feature cloud crítica

## Definiciones

| Término | Definición |
|---------|-----------|
| **IaaS** | Tú administras todo menos hardware |

| **Pod** | Kubectl smallest unit (containers+storage/net) |
