# 12. Support (Supplementals) - Soporte y Ayuda

## Resumen del Capítulo
Azure proporciona múltiples opciones de soporte para ayudarte. Este capítulo cubre planes de soporte Azure, canales de ayuda, Microsoft Learn, y recursos comunitarios. **Nota**: Este capítulo es SUPPLEMENTAL, no es parte del examen AZ-900 actual (refreshed May 2022).

## Temas Principales

### 1. Azure Support Plans (Planes de Soporte)

#### a) Free Support (Gratuito)
- **Incluido** con todas las suscripciones Azure
- Acceso a documentación oficial
- Community support
- Service health monitoring
- **Limitación**: No hay soporte ticket directo

#### b) Developer Support (~$25/mes)
- Para: No-production environments
- Response time: < 24 horas (durante horas trabajo)
- Email-based support
- **Casos válidos**: 
  - Technical guidance
  - Breaking changes

#### c) Standard Support (~$100/mes)
- Para: Production workloads
- Response times:
  - Non-critical: < 24 horas
  - System impaired: < 4 horas
  - Production impaired: < 1 hora
- Phone y email support

#### d) Professional Direct Support (~$1000/mes)
- Para: Business-critical applications
- Response times:
  - Non-critical: < 15 minutos
  - System impaired: < 15 minutos
  - Production down: < 15 minutos
- Phone support 24/7
- Assigned Technical Account Manager (TAM)
- Proactive guidance

Severity 2 (High):

Severity 3 (Medium):

### La Documentación Oficial

```
Microsoft Docs (docs.microsoft.com/azure)

Contiene:
├─ API references (exacto, oficial)
├─ How-to guides (detallado)
├─ Architecture patterns (best practices)
├─ Code examples (copyable)
├─ Todos los servicios

Calidad: Excelente (revisado y mantenido por Microsoft)

Pero: "Es referencia, no tutorial"
└─ Útil si ya sabes qué buscar
└─ No teach "primeros pasos"

Vs. Learn:
└─ Learn = Tutorial (enseña paso a paso)
└─ Docs = Reference (responde preguntas específicas)
└─ Idealmente usar AMBOS
```

### Stack Overflow + Reddit + Comunidad

```
Stack Overflow:
├─ Tag: "azure"
├─ Q&A format (estructura clara)
├─ Comunidad masiva (alguien ha tenido tu problema)
├─ Quality: Variable (algunos expertos, algunos novatos)
└─ Ventaja: Respuestas tienden ser de expertos (reputation points)

Reddit (/r/azure, /r/webdev, etc):
├─ Informal (less formal than SO)
├─ Opiones reales de usuarios (no official)
├─ Respues a veces más rápida (less moderation)
├─ Quality: Mixed (depende del subreddit)
└─ Ventaja: "Humans talking to humans"

Usage Tips:
├─ SIEMPRE busca primero (tu pregunta fue asked 100x)
├─ Si preguntas, sé específico ("Error exacto es...")
├─ Proporciona context (qué intentaste, qué salió mal)
└─ Respuestas no garantizadas (best effort community)
```

```
1. LEARN (Microsoft Learn)

   └─ "Tengo profesional que me ayuda rápido"
```

### Caso Práctico: Debugging "Real"

Developer en producción: App is slow. Necesita solución rápida.

**Step 1: Documentación?**
```
Busca: "Azure VM performance optimization"
Docs dice: "Usa diagnostics extension..."
"Revisa CPU usage..."
OK, empezó debugging
```

**Step 2: Stack Overflow?**
```
Pregunta: "Azure VM slow, CPU 100%, no idea why"
Respuesta: "Instala diagnostics, check logs..."
Más clues
```

**Step 3: Still stuck después 2 horas?**
```
Si está en hora de trabajo → Create support ticket (Standard plan)
Response < 4 hours
Engineer reviews logs, dice "tu indexed query es lento, try THIS"
```

**Step 4: Fixed!**

Sin support plan: "Stuck 1 week, business suffering"
Con support plan: "Resolved en horas, minimal downtime"
