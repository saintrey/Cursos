# 11. Pricing (Precios)

## Resumen del Capítulo
Entender cómo funciona el pricing en Azure es crítico. Este capítulo cubre modelos de precios, factores que afectan costos, y herramientas para estimar y optimizar gasto. Este capítulo **SÍ es parte** del examen AZ-900.

## Temas Principales

### 1. Modelos de Precios Azure

#### a) Pay-as-You-Go (Paga Conforme Usas)
- **Modelo default**
- Pagas solo por recursos que uses
- **Ventaja**: Flexibilidad, sin compromisos
- **Desventaja**: Predecibilidad menor

#### b) Reserved Instances (Instancias Reservadas)
- Compra computación con descuento
- Compromiso: 1 año o 3 años
- **Descuento**: 30-70% vs pay-as-you-go
- **Ventaja**: Ahorro significativo
- **Desventaja**: Menos flexible

#### c) Spot Pricing (Precios Spot)
- Usa capacidad Azure sobrante
- Muy barato (hasta 90% descuento)
- **Ventaja**: Costo muy bajo
- **Desventaja**: Azure puede terminar si necesita capacidad

#### d) Hybrid Benefit (Descuento Híbrido)
- Reutiliza licencias on-premises
- **Ventaja**: Reduce costos si ya tienes licenses
- Software Assurance required
- Aplica a: Windows, SQL Server, etc.

### 2. Factores que Afectan Precios

#### a) Compute

**Virtual Machines:**
```
Precio determina por:
- CPU (cores)
- RAM (memoria)
- Storage (disco)
- Tipo de OS (Windows vs Linux)
    → Windows es más caro
- Region (geografía)
```

**App Services:**
```
Precio por App Service Plan:
- Free/Shared (limitado)
- Basic (bajo costo)
- Standard (aplicaciones production)
- Premium (dedicado, máximo performance)
```

#### b) Storage

```
Blob Storage:
- Hot tier: Acceso frecuente (caro)
- Cool tier: Acceso infrequente (más barato)
- Archive tier: Acceso muy raro (muy barato)

Disk Storage:
- HDD: Bajo
- Standard SSD: Medio
- Premium SSD: Alto
- Ultra: Muy alto
```

#### c) Database

```
Cosmos DB:
- Costoso (pay for throughput)
- Global replication = más costo
- Pero rápido y flexible

Azure SQL:
- Serverless (por request)
- Provisioned (capacidad fija)

MySQL/PostgreSQL:
- Más barato que SQL
- Por compute + storage
```

#### d) Networking

```
Virtual Networks: GRATIS
Load Balancing: Bajo costo
VPN Gateway: Medio
ExpressRoute: Más caro (conexión dedicada)
CDN: Por data transferida
```

### 3. Cálculo de Costos

#### a) Variables Principales

```
Costo Total = 
  (Resource Type) × 
  (Quantity) × 
  (Hours/Duration) × 
  (Region Multiplier) × 
  (Discounts Applied)
```

#### b) Ejemplo: VM Simple
```
Windows Server 2019 (Standard D-series)
  - D2s v3: 2 vCPU, 8GB RAM
  - Region: East US
  - 730 horas al mes
  - Cost: ~$100/mes (aproximado)

Si corres 24/7 por mes:
  $100 × 1 = $100

Si corres solo 8 horas/día (10 días):
  $100 × (80/730) = ~$11
```

#### c) Ejemplo: Storage
```
Blob Storage (100GB, Hot tier, GRS)
  - Storage: ~$5/mes
  - Replication overhead: minimal
  - Total: ~$5/mes
  
Si Archive tier:
  - Storage: ~$0.50/mes
  - 90% cheaper pero latencia alta
```

### 5. Estrategias de Optimización de Costos

#### a) Right-Sizing
```
Problema: VM sobredimensionada
Solution: Usar tamaño correcto
Ahorro: 20-50%

Ejemplo:
  Tienes D4 VM (4 vCPU)
  Pero usa 20% CPU
  → Downsize a D2 VM
  → Mitad del costo
```

#### b) Shutdown no-usado
```
Dev VMs corriendo 24/7
  → Apaga noches/weekends
  Ahorro: 50-70%
```

#### c) Reserved Instances
```
1 año commitment:
  → 30% off vs pay-as-you-go

3 años commitment:
  → 50-70% off

Mejor para: Predictable workloads
```

#### d) Spot Instances
```
Batch processing jobs
  → Usa Spot VMs
  → 90% more barato
  → OK si interruptions acceptable
```

#### e) Auto-Shutdown
```
Automáticamente apaga resources
  a hora específica
  → Dev/test environments
  → Reduce costos sin esfuerzo
```

#### f) Monitor y Alert
```
Azure Alerts:
  "Si presupuesto > $1000/mes"
    → Envía notificación
  
  Permite re-actuar antes de surprise bill
```

### 6. Factores Regionales

#### a) Precios Varían por Región
```
East US:     Baseline
West US:     Medio
Europe:      Medio-Alto
Asia Pacific: Alto

Diferencia: 20-30% común
```

#### b) Decisión: Precio vs Latencia
```
Cheapest region ≠ Best region
  - Si server en South Africa
  - Pero users en USA
  - Latencia alta
  
Balance needed:
  - Costo razonable
  - Latencia aceptable
  - Features disponibles
```

### 8. Tipos de Costos

#### a) Direct Costs (Costos Directos)
- VM usage
- Storage usage
- Database throughput
- Fácilmente identificables

#### b) Indirect Costs (Costos Indirectos)
- Networking (data transfer out)
- Redundancy (replicación)
- Premium features
- Menos obvios

#### c) Hidden Costs (Costos Ocultos)
```
Comunes:
- Data egress (salida de datos)
  → $0.12/GB típicamente
  
- Premium tiers activadas by default
  
- Over-provisioned resources
  
- Abandoned resources
  → Still charged though no usage
```

### 11. TCO: On-Premises vs Cloud

```
ON-PREMISES (5 años):
├─ Hardware: $50,000
├─ Software licenses: $30,000
├─ Staff (admin/maintain): $100,000/año × 5 = $500,000
├─ Power/cooling: $20,000/año × 5 = $100,000
├─ Real estate: $50,000
└─ TOTAL: ~$730,000

AZURE (5 años):
├─ VMs + Services: $1000/mes × 60 months = $60,000
├─ Bandwidth/data: $5000/año × 5 = $25,000
└─ TOTAL: ~$85,000

AHORRO: ~$645,000 (88% reduction)
```

## 📚 Conclusión y Síntesis Profunda - Capítulo 11: Pricing

### ¿Qué Aprendiste Realmente?

**Pricing NO es un accidente. Es un modelo económico completo.**

Aprendiste que "cloud es más barato" es VERDAD PERO SOLO si entiendes la economía. Una aplicación mal diseñada en Azure puede costar 10x más que en on-premises porque está corriendo 100 VMs innecesarias.

### La Ecuación Fundamental

```
SIN CLOUD:
Costo = Equipamiento + Staff + Potencia + Espacio + Licenses
        = $100,000+ year (fijo, no importa si usas o no)

CON CLOUD:
Costo = Recursos actualmente consumidos
      = Escalabilidad de costo con uso
```

La diferencia es RADICAL. En on-premises pagas por capacidad. En cloud pagas por consumo.

### Pay-as-You-Go: La Flexibilidad

```
Mes 1: Startup pequeño = $500/mes (5 VMs)
Mes 6: Tráfico sube 10x = $5,000/mes (50 VMs)
Mes 8: Reducen feature = $1,000/mes (10 VMs)
Mes 12: De nuevo sube = $10,000/mes (100 VMs)

Total año: $50,000 (promedio $4,200/mes)

Si on-premises:
└─ Provisionar para 100 VMs = $120,000/año
└─ Pero usaste promedio 35 VMs
└─ $85,000 desperdiciado
```

**Eso es el poder.** El costo crece CON el negocio, no antes.

### Reserved Instances: El Trade-Off

```
Pay-as-you-go:
$5,000/mes × 12 = $60,000/año (pero flexible)

Reserved 1-year:
$3,000/mes × 12 = $36,000/año (40% off, pero COMPROMETIDO)

Reserved 3-year:
$2,500/mes × 12 = $30,000/año (50% off, pero TRIPLE comprometido)
```

**¿Cuándo?**

```
Patrón de tráfico es 100% predecible:
└─ Producción 24/7 monolítica
└─ Reserved = Obvious

Patrón es variable/experimental:
└─ Startup con tráfico impredecible
└─ Pay-as-you-go = Mejor
```

### Spot Pricing: El Juego Económico

Azure: "Tengo capacidad extra de servidores. ¿Quién la usa con descuento?"

```
Normal: $100/VM
Spot: $10/VM

Pero: "Si necesito capacidad, boto tu VM"

Casos válidos:
├─ Batch processing jobs (análisis nocturnos)
├─ Renderizado (puede esperar)
├─ Testing (si se cae, re-run)

NO válido:
├─ Production web app (usuarios ven downtime)
```

### El Costo Oculto: Data Transfer

```
Muchos ignoran esto y shockeado con factura

Scenario:
├─ Run análisis eneúje: Envía resultado a on-prem
├─ 1 TB data transfer = $120+ (a $0.12/GB)
└─ "¿Por qué factura tan alta?"

Solution:
├─ Procesa en Azure, guarda en Azure
├─ Evita transferencia external
└─ O menos transfers
```

**Lección:** Data que sale Azure es CARO. Mantén dentro si puedes.

### Tagging: Facturación por Equipo

```
Sin tagging:
├─ Facturación = Empresa total
├─ ¿Quién gastó qué? No sé
└─ Cada equipo adivina

Con tagging:
├─ VMs tag: team=frontend, environment=prod, cost-center=sales
├─ Factura: Frontend gastó $5000, Backend $3000
├─ Cada equipo ve su costo
├─ Motivación: Optimizar, no desperdiciar

Resultado: Cultura de costo-awareness.
```

### Advisor & Automation: El Asistente

Azure Advisor automáticamente:

```
Scan VMs:
└─ "Esta VM idle 90% del tiempo"

Recommendation:
└─ "Resize a más pequeña (40% menos costo)"
└─ O "Apágala y re-crearla si necesitas"

Deploy:
└─ 1 Click
```

Muchas compañías ahorran 30-40% solo siguiendo Advisor.

### Caso Práctico: La Migración

TechCorp migra de on-prem a Azure:

**On-Premises (5 años):**
```
Datacenter: $200K (setup)
Staff (3 admins @ $100K/year): $1.5M
Electricity: $50K/year = $250K
Hardware maintenance: $50K/year = $250K
Network/Security appliances: $100K
────────────────────────
TOTAL: $2.35M+++ (y no escala bien)
```

**Azure (5 años):**
```
VMs computación (avg $1000/mes): $60K

Support Engineer (0.25 FTE @ $100K): $125K
────────────────────────
TOTAL: $235K (10x menos!)
```

**Ahorro: $2.1M en 5 años.**

### Lo Que Deberías Saber AHORA

1. "¿Cuál es el modelo default?"
   **Respuesta:** Pay-as-you-go (flexible, pagas por consumo)

2. "¿Cómo reduzco factura?"
   **Respuesta:** Reserved (40% off) O Spot (90% off si toleras interruption) O Right-sizing

3. "¿Por qué mi factura exploto?"
   **Respuesta:** Recursos ociosos, data egress, premium tiers

4. "¿Cuándo Azure es más barato ON-prem?"
   **Respuesta:** Siempre, si usas bien. Staff savings es 80% del ahorro.
